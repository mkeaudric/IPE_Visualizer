Jika kamu ingin mengeksplorasi atau menerapkan bahasa lain untuk membuat program ini menjadi satu ekosistem yang lebih utuh, 
modern, atau otomatis, berikut adalah beberapa bahasa yang sangat masuk akal untuk ditambahkan beserta perannya:

1. Python (Untuk Automasi, Pembersihan Data, & Batch Testing)
   Python adalah bahasa pendamping paling populer untuk program berbasis Java atau C++. 
   Dalam konteks program kamu, Python bisa ditaruh di lapisan luar (wrapper).
   Perannya: * Daripada kamu mengetik manual 15 titik di terminal Java, kamu bisa membuat skrip Python pendek untuk 
   me-generate ratusan koordinat acak secara otomatis ke dalam file .txt.
   Skrip Python tersebut kemudian bisa otomatis mengeksekusi program Java kamu (java VisualizerIPE < input.txt), 
   mengambil file .ipe hasilnya, dan langsung membukanya di aplikasi IPE.
   Kamu juga bisa pakai Python untuk melakukan benchmark waktu eksekusi (mengukur performa linear O(n) Welzl dengan 
   membuat grafik pertumbuhan lewat pustaka Matplotlib).

2. Bash / Shell Script (Untuk Otomasi Pipeline di Terminal)
   Jika kamu pengguna Linux atau macOS (atau Git Bash di Windows), kamu bisa menggunakan Bash script (.sh).
   Perannya:Menghubungkan alur kerja program dari hulu ke hilir. 
   Kamu cukup menjalankan satu perintah ./run_visualizer.sh, dan skrip tersebut akan otomatis:
   - Melakukan compile semua file Java (javac *.java).
   - Menjalankan program utama dengan input otomatis.
   - Memindahkan file hasil.ipe ke dalam folder arsip tugas besar.

3. JavaScript / TypeScript (Jika Ingin Membuat GUI Berbasis Web)
   Saat ini kamu menggunakan terminal (Scanner sc) untuk input dan aplikasi pihak ketiga (IPE) untuk melihat hasilnya. 
   Bagaimana kalau kamu ingin visualisasinya muncul interaktif di browser internet?
   Perannya:Kamu bisa membuat frontend sederhana menggunakan HTML/CSS dan JavaScript.Java milikmu tetap berjalan di latar belakang 
   sebagai server (bisa pakai Spring Boot yang pernah kamu pelajari) untuk menghitung algoritma Welzl.
   JavaScript di browser akan bertugas menangkap klik mouse user (misal user nge-klik layar untuk naruh titik), mengirim koordinatnya 
   ke Java, lalu menggambar lingkaran hasilnya secara real-time di elemen <canvas> HTML5 tanpa perlu membuka aplikasi IPE lagi.

4. Makefile (Pustaka Otomasi Build)
   Meskipun bukan bahasa pemrograman murni, sintaks Makefile sering digunakan di proyek-proyek informatika tingkat universitas.
   Perannya:Mengatur dependensi kompilasi. Kamu cukup menulis perintah make run di terminal, dan Makefile akan tahu file mana saja 
   yang berubah dan perlu di-compile ulang, sehingga menghemat waktu pengerjaan tugas besar secara signifikan.
   
Kesimpulan Arsitekturnya Jika Digabungkan:
- Core Logic (Otak): Java (menghitung Welzl SEC).
- Data Visual (Output): XML IPE (format grafis vector).
- Automation (Pembantu): Python / Bash (mengurus supply data input dan eksekusi file).
- Interface (Tampilan, jika dikembangkan): JavaScript (interaksi klik titik di web).


Di dunia kerja nyata, praktik polyglot ini sangat lumrah. Misalnya, sebuah aplikasi web modern bisa menggunakan 
JavaScript di front-end, Java/Go di back-end untuk komputasi berat, Python untuk tim data science, dan SQL untuk berkomunikasi dengan database.
